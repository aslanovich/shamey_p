# Сайт психолога: Next.js + Cal.com

Лендинг с онлайн-записью: клиент видит только свободные слоты (синхронизация с Google Calendar), оставляет комментарий при записи и автоматически получает ссылку на видеовстречу.

## Шаг 1. Запуск локально

```bash
npm install
npm run dev        # http://localhost:3000
```

Положи портрет в `public/portrait.png` (фон у фото лучше сделать прозрачным или белым — градиент внизу «растворит» его в страницу, как на текущем лендинге).

## Шаг 2. Настройка Cal.com (~15 минут)

1. Зарегистрируйся на [cal.com](https://cal.com) (бесплатный тариф покрывает всё нужное).
2. **Apps → Google Calendar → Install.** После подключения Cal.com будет проверять занятость и скрывать занятые слоты, а новые записи — создавать события в твоём календаре.
3. **Apps → Google Meet → Install** и выбери Meet как локацию по умолчанию — ссылка на видеовстречу будет создаваться и уходить клиенту автоматически. (Zoom подключается так же.)
4. Создай **Event Type** «Консультация», 50 минут. Здесь же настрой:
   - **Availability** — рабочие часы;
   - **Limits** — буфер 10–15 минут между сессиями, максимум записей в день, запрет записи менее чем за N часов;
   - **Booking questions** — добавь поле «С чем хотите прийти?» (тип Long text, необязательное). Это тот самый комментарий; его видишь только ты — в письме и в описании события в календаре.
5. Создай второй Event Type «Знакомство», 20 минут — бесплатная ознакомительная встреча.
6. **Workflows** — включи напоминания клиенту за 24 часа и за 1 час (минус no-show).
7. Скопируй ссылку события вида `имя/session` и вставь в `components/Booking.tsx` → `CAL_LINK`.

Если на лендинге нужны обе кнопки (знакомство и консультация) — продублируй компонент `Booking` со вторым `calLink` и другим `namespace`, либо сделай переключатель-табы.

## Шаг 3. Деплой

Проще всего — [Vercel](https://vercel.com): импортируешь репозиторий с GitHub, деплой происходит автоматически при каждом пуше. Бесплатного тарифа достаточно. Свой домен подключается в настройках проекта.

## Шаг 4. Когда практика вырастет

- **Оплата при записи** — в Cal.com подключается Stripe (Apps → Stripe): клиент платит в момент бронирования, no-show почти исчезают.
- **Self-hosted Cal.com** — если клиенты из ЕС и важно, где лежат их данные: Cal.com открытый, разворачивается на своём сервере.
- **Своя страница /privacy** — файл `app/privacy/page.tsx`, ссылка из футера уже стоит. Для записи к психологу политика конфиденциальности обязательна.

## Почему не свой календарь на Google Calendar API

Можно (Next.js API route + freebusy-запрос + Supabase), но это 5–10 дней работы и на тебе будут таймзоны, гонки за один слот, письма и переносы. Cal.com решает всё это из коробки — разумно начать с него и переписать на своё, только если упрёшься в ограничения.
